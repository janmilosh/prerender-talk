Why we might still need this:

  • If SEO is critical for your app
  • If your JavaScript is too complex
  • To be indexed by more search engines
  • Using JS by crawlers is still new.

There are many methods for serving pre-rendered pages.

The point here is to show one way, and that it's quick and easy.

For this presentation:

  • AngularJS app (with firebase backend), TaskMe.us
  • Apache server (Digital Ocean)
  • Prerender.io to serve rendered pages to webcrawlers.

How it works:

  • In our routing, change the hashPrefix to '#!'.
  • Search engines change the request to use '_escaped_fragment_' instead of '#!'
  • '_escaped_fragment_' requests are served pre-rendered HTML.

How it works, cont:

  • Prerender.io uses a headless browser to render the pages.
  • Prerender.io takes care of serving the pages.
  • For Apache, the .htaccess file tells the page location.

Step 1:

Turn off HTML5 mode
and
set the hash prefix.

Step 2:

Be sure to change href's to '#!'.

Step 3

Create Prerender.io account.

Put token and url in code in .htaccess
