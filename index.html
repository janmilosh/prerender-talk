SEO Problem:

Indexing pages for single-page apps - where pages are rendered in the browser with JavaScript

Solution:

Pre-render your pages.

Let googlebots and others know where to find them.

Why we might still need this:

  • If SEO is critical for your app
  • If your JavaScript is too complex
  • To be indexed by more search engines
  • Using JS by crawlers is still new.

There are many methods for serving pre-rendered pages.

The point here is to show one way, and that it's quick and easy.

For this presentation:

  • AngularJS app (with firebase backend), TaskMe.us
  • Apache server (Digital Ocean)
  • Prerender.io to serve rendered pages to webcrawlers.

How it works:

In our routing, by changing the hashPrefix to '#!' instead of '#', search engines change the request to use '_escaped_fragment_' instead of '#!'

When requests come with an '_escaped_fragment_' querystring, a pre-rendered HTML response is served (instead of your app).

How it works, cont:

The Prerender.io service takes care of rendering and serving your pages.

For Apache, the .htaccess file tells crawlers where to find the pages.
